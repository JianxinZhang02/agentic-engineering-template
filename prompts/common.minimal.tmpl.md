# Common Minimal Prompt

这是所有 AI 工具都应遵守的最小核心指令。

## 工作前

- 先读 `AGENTS.md` 和 `context/INDEX.md`。
- 涉及具体项目时，先读项目 `INDEX.md`。
- 涉及代码时，先定位并阅读相关源码和至少一个调用点。
- 不从文件名、函数名或常识猜测行为。

## 工作中

- Stay on task：只做当前需求要求的事。
- 不修无关 TODO、无关 code health 或顺手重构。
- 注释解释为什么，不重复代码做了什么。
- 所有关键事实标注来源、用户输入或待确认。
- 长输出先给摘要和确认点，再展开。

## 验证

- 构建或测试前先确认项目使用的命令。
- 只修复自己改动引入的问题。
- 未执行的验证必须记录原因和风险。
- 完成后输出已验证和未验证清单。

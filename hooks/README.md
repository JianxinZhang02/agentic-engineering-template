# Hooks

Hook 是事件驱动的自动化约束。不是所有工具都支持 Hook；不支持时，应把本目录作为执行前后的检查清单。

## Hook 类型

| 类型 | 时机 | 用途 |
| --- | --- | --- |
| PreToolUse | 工具执行前 | 拦截危险操作、提醒确认 |
| PostToolUse | 工具执行后 | 快速检查输出、记录状态 |
| Stop | 每轮响应后 | 保存摘要、检查调试残留、提示沉淀 |
| SessionStart | 会话开始 | 加载项目状态和需求上下文 |
| PreCompact | 上下文压缩前 | 保存当前进度 |
| SessionEnd | 会话结束 | 写入会话摘要和未完成事项 |

## 推荐 Hook

| Hook | 行为 | 级别 |
| --- | --- | --- |
| 受保护分支拦截 | 阻止直接向主分支提交或推送 | block |
| 提交前质量检查 | 检查格式、测试记录、密钥、调试日志 | block/warn |
| 长命令提醒 | 构建、测试、服务启动前提醒保存日志 | warn |
| 文档创建提醒 | 新增 Markdown 时提醒选择正确目录 | warn |
| 上下文保存 | 会话结束或压缩前保存当前需求状态 | warn |
| 知识沉淀提醒 | 需求进入验收时检查 `notes.md` | warn |

## 工具适配

- Claude Code：可映射为原生 hooks。
- Codex：作为人工/指令式门禁执行。
- Cursor/OpenCode：按各自事件系统映射。
- WorkBuddy：只暴露高层确认点，不暴露内部 hook 细节。

## 设计规则

- Hook 必须可解释、可禁用、可分级。
- Hook 不应依赖个人本地路径。
- 阻塞型 Hook 只用于高风险或不可逆操作。
- Hook 输出应包含修复建议。

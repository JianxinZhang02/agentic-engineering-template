# Testing Standard

## 测试目标

测试记录必须让审查者知道：什么已经验证，什么没有验证，剩余风险是什么。

## 测试层级

- 单元测试：验证函数、类、模块的局部行为。
- 集成测试：验证模块之间的数据流和契约。
- 端到端测试：验证用户可见工作流。
- 手工验证：记录不可自动化或暂未自动化的检查。

## 测试计划必须包含

- 需求对应的验收标准。
- 正常路径。
- 异常路径。
- 边界条件。
- 回归风险。
- 不测试项和原因。

## 测试结果记录

```text
已执行：
- 命令/步骤：
  结果：

未执行：
- 项目：
  原因：
  风险：
```

## Agent 约束

- 未执行的测试不能写成已通过。
- 不能用“看起来没问题”替代验证。
- 修复测试失败前，必须先理解失败原因。
- 如果环境缺失，应记录缺失项和建议补救方式。

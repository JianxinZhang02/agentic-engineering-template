# Metrics

模板的目标不是堆叠更多 Agent 或 Skill，而是提升 AI 协作研发的可靠性和复利。以下指标用于判断系统是否真的变好。

## 使用指标

| 指标 | 含义 | 记录位置 |
| --- | --- | --- |
| 需求启动耗时 | 从复制模板到需求定义初稿的时间 | `requirements/{id}/notes.md` |
| 上下文命中率 | AI 输出中有来源标注的关键事实占比 | `review-report.md` |
| 门禁拦截数 | 阶段切换前发现的问题数 | `review-report.md` |
| 测试记录完整度 | 已执行/未执行验证是否清楚 | `test-plan.md` |
| 知识沉淀数 | 每个需求沉淀的有效知识条目数 | `acceptance.md` |

## 质量指标

- 需求返工次数。
- 代码审查阻塞问题数。
- AI 编造事实次数。
- 重复问题再次出现次数。
- `context/` 过时知识修复次数。

## 工具指标

- Command 触发频率。
- Skill 使用频率。
- Agent 输出被采纳比例。
- Hook 拦截/提醒次数。
- Eval 通过率。

## 最小记录格式

```text
日期：
需求：
触发动作：
结果：
是否节省时间：
是否发现问题：
是否需要沉淀：
```

指标先保持轻量，不需要一开始建设 dashboard。

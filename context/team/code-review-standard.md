# Code Review Standard

## 审查目标

代码审查的目标是发现会影响正确性、安全性、可维护性、性能、测试可靠性的具体问题。

## 必查维度

- 需求一致性：实现是否覆盖需求和验收标准。
- 正确性：边界条件、错误处理、并发、状态流转是否正确。
- 安全性：鉴权、权限、注入、敏感信息、依赖风险。
- 可维护性：命名、复杂度、重复代码、模块边界。
- 测试充分性：是否覆盖核心路径、异常路径和回归风险。
- 兼容性：数据、接口、配置和部署是否兼容。

## 输出格式

```text
总体结论：通过 / 需修改 / 阻塞
阻塞问题：
主要问题：
建议优化：
已验证事项：
未验证事项：
```

## 严重级别

- `blocking`：必须修复，否则不能合入。
- `major`：建议本次修复，影响质量或稳定性。
- `minor`：非阻塞优化。
- `question`：需要作者确认的疑问。

## Agent 约束

- 只报告基于 diff、上下文或规范能够支撑的问题。
- 不输出泛泛而谈的建议。
- 不要求无关重构。
- 未看到测试结果时，必须明确标注未验证。

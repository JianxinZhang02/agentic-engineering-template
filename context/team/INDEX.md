# Team Context Index

团队级知识适用于所有项目，是 Agent 工作时默认遵守的基础规范。

## 文档路由

- Git 分支、提交、PR：读取 `git-workflow.md`。
- 工程原则、变更边界、设计取舍：读取 `engineering-principles.md`。
- AI 责任边界、安全和隐私：读取 `ai-policy.md`。
- 代码审查、风险分级、报告格式：读取 `code-review-standard.md`。
- 测试策略、测试记录、未验证说明：读取 `testing-standard.md`。
- 知识沉淀、索引维护、规范瘦身：读取 `knowledge-management.md`。
- 是否新增或调整工具：读取 `tooling-governance.md`。
- 质量、复利和工具价值度量：读取 `metrics.md`。

## 默认门禁

- 事实门禁：关键事实必须有来源、用户输入或待确认标记。
- 阶段门禁：当前阶段产物完整后才能进入下一阶段。
- 追溯门禁：需求、设计、实现、测试、验收必须互相对应。
- 代码门禁：代码审查覆盖正确性、安全性、可维护性、测试充分性。
- 知识门禁：需求完成后检查是否有知识需要沉淀。
- 安全门禁：敏感场景必须执行安全审查。
- 验证门禁：合入前必须记录已执行和未执行验证。

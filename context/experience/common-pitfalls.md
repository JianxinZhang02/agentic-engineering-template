# Common Pitfalls

## AI 协作常见错误

| 错误 | 影响 | 避免方式 |
| --- | --- | --- |
| 未读上下文就写需求 | 容易编造项目事实 | 先读 `context/INDEX.md` 和项目索引 |
| 把待确认项写成事实 | 误导后续设计和实现 | 使用 `来源/用户输入/待确认` 标注 |
| 一次性输出长文档 | 人类难以及时纠偏 | 先输出 3-5 条确认点 |
| AGENTS.md 膨胀 | 指令过载导致 Agent 走神 | 入口轻量，细节下沉 |
| 需求完成后不沉淀 | 下次继续重复解释 | 验收阶段执行知识门禁 |

## 规范维护常见错误

- 为偶发问题创建永久规则。
- 新增 Skill 但没有真实复用频率。
- 修改 `context/` 但忘记更新 `INDEX.md`。
- 过时知识未清理，导致 Agent 基于错误事实工作。

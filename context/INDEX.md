# Context Index

`context/` 是本仓库的公共记忆。Agent 必须先读本索引，再决定读取哪些具体文档。

## 目录

- `team/`：团队级通用规范，适用于所有项目。
- `project/`：项目级业务、架构、接口、术语和经验。
- `experience/`：跨项目可复用的踩坑经验和通用模式。

## 读取顺序

1. 如果任务涉及研发流程、Git、评审、测试或知识沉淀，读取 `team/INDEX.md`。
2. 如果任务涉及具体项目，读取 `project/{project}/INDEX.md`。
3. 如果任务涉及反复出现的问题、AI 易错点或跨项目经验，读取 `experience/README.md`。
4. 只读取与当前任务强相关的文档，不要把整个 `context/` 全量塞入上下文。

## 事实标注规则

任何写入需求、设计、评审或知识库的关键事实，都必须标注来源：

- `来源: context/...`：来自仓库文档。
- `来源: 用户输入`：来自当前对话或需求原始描述。
- `待确认`：尚无可靠来源，需要人类确认。

## 维护规则

- `context/` 中的知识必须可复用。
- 过时知识比没有知识更危险，发现后必须更新或标记废弃。
- 新知识优先进入需求 `notes.md`，验收阶段再沉淀到长期位置。
- 不把一次性讨论、临时日志、未验证猜测写入长期知识库。

# Skill: verification-loop

## 触发条件

- 完成功能、修复、重构或文档规范改动后。
- 创建 PR 前。
- 用户要求验证、回归或质量门禁。

## 输入

- 当前 diff。
- 关联需求目录。
- `context/team/testing-standard.md`
- 项目构建、测试、lint、格式化命令。

## 输出

```text
验证结论：
已执行：
失败项：
未执行：
风险：
下一步：
```

## 验证阶段

1. 构建或静态检查。
2. 类型检查或编译。
3. lint 和格式化。
4. 测试套件。
5. 安全和密钥快速检查。
6. diff 自审。
7. 追溯检查。

## 规则

- 先确认项目命令，不猜测。
- 失败后停止并修复关键失败。
- 只修复当前改动引入的问题。
- 未执行项必须说明原因和风险。

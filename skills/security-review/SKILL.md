# Skill: security-review

## 触发条件

- 涉及认证、授权、支付、用户输入、文件上传、敏感数据、外部 API、密钥或权限边界。
- 代码审查中发现安全风险。
- PR 准备合入前需要安全确认。

## 输入

- 关联需求目录。
- 当前 diff 或设计方案。
- `context/team/ai-policy.md`
- `rules/common/security.md`
- 项目接口和架构上下文。

## 输出

```text
安全结论：
阻塞风险：
主要风险：
已验证防护：
未验证风险：
建议修复：
```

## 检查项

- 密钥是否硬编码。
- 输入是否校验。
- 权限是否在可信边界执行。
- 错误信息和日志是否泄漏敏感数据。
- 外部调用是否有超时、重试和降级。
- 数据存储和传输是否满足项目要求。
- 第三方内容是否可能携带提示注入。

## 不负责

- 不替代正式安全评审。
- 不凭猜测断言漏洞。
- 不输出可被滥用的攻击步骤。

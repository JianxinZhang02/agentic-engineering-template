# Skill: context-collection

## 触发条件

- 开始需求定义、方案设计、代码审查或知识整理。
- 用户要求“先了解项目”“查上下文”“不要编造”。

## 输入

- `context/INDEX.md`。
- `context/team/INDEX.md`。
- 相关项目的 `context/project/{project}/INDEX.md`。
- 当前需求目录。

## 输出

```text
已读取：
关键事实：
缺口：
待确认：
建议下一步：
```

## 流程边界

1. 先读总索引。
2. 判断任务类型。
3. 读取强相关团队规范。
4. 读取相关项目索引。
5. 只读取强相关项目文档。
6. 记录事实来源和缺口。

## 禁止事项

- 不全量读取 `context/`。
- 不用互联网资料替代项目事实。
- 不把相似项目经验直接套用到当前项目。

# Skill: requirement-lifecycle

## 触发条件

- 用户要新建、推进、回退、查看或验收需求。
- 用户提到“需求”“方案”“实现计划”“下一阶段”“验收”等生命周期动作。

## 输入

- 用户原始需求。
- 当前需求目录。
- `context/INDEX.md`。
- 相关项目的 `context/project/{project}/INDEX.md`。
- `templates/` 中的需求、设计、实现计划、测试计划模板。

## 输出

- 更新后的需求生命周期文件。
- 阶段门禁检查结果。
- 待确认问题。
- 下一步建议。

## 流程边界

1. 需求启动：创建或识别需求目录，保留原始输入。
2. 上下文收集：调用 `context-collection` 思路，记录来源和缺口。
3. 需求定义：先输出确认点，再生成正式需求。
4. 方案设计：基于项目上下文生成设计和实现计划。
5. 实现与测试：记录测试计划、已验证和未验证事项。
6. 验收与沉淀：整理 `notes.md`，调用 `knowledge-curation` 思路。

## 门禁

- 事实门禁。
- 阶段门禁。
- 追溯门禁。
- 测试门禁。
- 知识门禁。

## 不负责

- 不直接决定业务取舍。
- 不绕过人类确认进入关键阶段。
- 不创建无关项目知识。

# Skill: code-review

## 触发条件

- 用户要求审查代码、PR、diff、提交或实现方案。
- 需求进入实现完成后的质量检查阶段。

## 输入

- 当前 diff 或变更说明。
- 需求目录中的 `requirement.md`、`design.md`、`implementation-plan.md`、`test-plan.md`。
- `context/team/code-review-standard.md`。
- 相关项目上下文。

## 输出

```text
总体结论：
阻塞问题：
主要问题：
建议优化：
已验证事项：
未验证事项：
```

## 审查维度

- 需求一致性。
- 正确性。
- 安全性。
- 可维护性。
- 测试充分性。
- 兼容性。

## 流程边界

1. 先理解需求和设计。
2. 再审查 diff。
3. 问题必须绑定具体证据。
4. 不确定的问题以 question 输出。
5. 未验证事项必须明确记录。

## 不负责

- 不做无关重构建议。
- 不把个人偏好当作阻塞问题。
- 不替代人类最终合入判断。

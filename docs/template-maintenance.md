# Template Maintenance

模板维护的目标是让后续项目更容易开始、更少重复踩坑，而不是追求目录越来越多。

## 变更入口

- 改 AI 行为入口：优先更新 `AGENTS.md` 或 `prompts/`。
- 改团队规范：更新 `context/team/`。
- 改工具化能力：更新 `skills/`、`commands/`、`agents/`。
- 改可安装规则：更新 `rules/`。
- 改自动化提醒：更新 `hooks/`。
- 改外部连接：更新 `mcp-configs/`。
- 改评估：更新 `eval/`。

## 每月维护清单

- [ ] `AGENTS.md` 是否仍是入口索引。
- [ ] `context/INDEX.md` 是否准确。
- [ ] 是否存在过时项目事实。
- [ ] 是否有低频规则占用通用上下文。
- [ ] 新增 Skill 是否有真实复用。
- [ ] Eval 是否覆盖最近一次模板改动。
- [ ] 是否有本地路径、私有信息或密钥残留。

## 删除标准

满足任意一项即可考虑删除或降级：

- 30 天未触发。
- 每次都通过且不产生价值。
- 可用自然语言低成本替代。
- 与其他规则重复。
- 引起误报或上下文噪音。

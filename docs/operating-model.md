# Operating Model

本模板的运行模型是“人类决策 + AI 执行和整理 + Git 版本化知识”。

## 角色分工

| 角色 | 负责 |
| --- | --- |
| 人类负责人 | 目标、范围、业务取舍、最终验收 |
| 主 Agent | 上下文检索、任务协调、状态维护 |
| Skill | 可复用领域能力 |
| Agent | 独立审查、探索和维护 |
| Hook | 机械化提醒或拦截 |
| Eval | 行为回归验证 |

## 标准循环

```text
需求输入
-> 上下文收集
-> 关键确认点
-> 需求和设计
-> 实现与测试
-> 审查与验证
-> 验收
-> 知识沉淀
-> 模板和规则迭代
```

## 维护节奏

- 每个需求结束：整理 `notes.md`。
- 每周：检查未关闭的待确认项。
- 每月：执行 tool stocktake 和规范瘦身。
- 每次模板重大改动：复跑 `eval/`。

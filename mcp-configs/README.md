# MCP Configs

MCP 配置用于连接外部系统。模板只保留配置样例和风险边界，不提交真实密钥或个人配置。

## 适合 MCP 的能力

- 需求系统读取。
- 企业知识库检索。
- GitHub/GitLab 查询。
- 测试平台触发。
- 浏览器或页面验证。
- 通知系统。

## 风险边界

- MCP 能力越强，越需要明确权限和审计。
- 写操作必须优先使用测试环境或沙箱。
- 敏感系统必须有人类确认点。
- 配置文件不得包含 token、cookie、密码或私钥。

## 文件

- `mcp-servers.example.json`：服务配置示例。
- `extensions.catalog.json`：可用扩展/MCP 的目录、owner、状态和风险摘要。

## 管理建议

- 先列出可用扩展，再安装或启用。
- 默认使用链接或引用，避免复制后版本漂移。
- 从项目级配置迁移到用户级配置时，要避免重复启用同一 MCP。
- 每个扩展必须有 owner 和风险说明。

## 沉淀原则

连接本身不是解决方案。真正有价值的是：

- MCP 配置。
- 相关上下文。
- Skill 编排。
- 门禁和验证。
- 经验沉淀。

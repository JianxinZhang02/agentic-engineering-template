# Command: requirement-next

## 目的

在进入下一阶段前执行门禁检查。

## 输入

- 当前需求目录。
- 当前阶段。

## 门禁清单

### 事实门禁

- [ ] 关键事实都有来源、用户输入或待确认标注。
- [ ] 待确认项没有写成事实。

### 阶段门禁

- [ ] 当前阶段产物完整。
- [ ] 下一阶段需要的输入已准备。

### 追溯门禁

- [ ] 需求点能对应设计点。
- [ ] 设计点能对应实现任务。
- [ ] 实现任务能对应测试项。

### 代码门禁

- [ ] 如果已有代码变更，已执行代码审查。
- [ ] 已记录已验证和未验证事项。

### 验证门禁

- [ ] 已按 `skills/verification-loop/SKILL.md` 执行或记录无法执行的原因。
- [ ] 构建、测试、lint、安全快速检查结果已写入 `test-plan.md` 或 `review-report.md`。

### 安全门禁

- [ ] 如果涉及认证、授权、用户输入、文件上传、敏感数据、支付或外部 API，已执行 `skills/security-review/SKILL.md`。
- [ ] 安全风险已标注阻塞、主要或待确认。

### 知识门禁

- [ ] 如果需求已完成，`notes.md` 已整理。
- [ ] 可复用知识已沉淀或明确无沉淀项。

## 输出

```text
当前阶段：
目标阶段：
通过项：
阻塞项：
待确认：
下一步：
```
